yarn link

在开发的时候，用Symlink来连接一个项目包文件夹。

在开发阶段的项目包，可以被其他项目所依赖。因为还在开发阶段，项目包并没有被发布，通常用这种方式在其他项目中来测试新功能或者debug。

工作流需要执行两个命令：

yarn link (在link目标文件夹中执行)

link目标文件夹。举例，如果你现在准备debugreact-relay中的第一个问题，react-relay依赖了react，希望能够使用本地的react进行debug，那么你需要进入到react项目文件夹中，执行yarn link命令。

yarn link [package...]

在你需要link的地方，运行yarn link [package]。下面的例子中，在react-relay项目的文件夹中执行yarn link react命令去链接之前已经link过的本地react。

完整的示例，假设react文件夹和react-relay文件夹是紧挨着的：

$ cd react
$ yarn link
yarn link vx.x.x
success Registered "react".
info You can now run `yarn link "react"` in the projects where you want to use this module and it will be used instead.

$ cd ../react-relay
$ yarn link react
yarn link vx.x.x
success Registered "react".

这将会创建一个名为react-relay/node_modules/react的符号链接，来link本地的react文件夹。

运行yarn unlink或者yarn unlink [package]，可以取消之前的link操作。更多请见：

  • yarn unlink：取消已经link的依赖
