yarn tag

对包的tag执行增加、删除和查看的操作

什么是tag?

tag就是为你发布的版本打上标签，这样使用包的用户能够直接通过tag安装，而不需要输入版本号，这样方便很多。

例如，如果你有一个stable版本的发布，和一个canary版本的发布，可以执行下面的命令：

yarn add your-package-name@stable
yarn add your-package-name@canary

对tag的命名，每种tag对应不同的版本，这是有约定俗成的规定：

  • latest: 当前的版本
  • stable: 当前最新的稳定版本，如果你没有设置Long-term Support (LTS) 版本，那么这个tag与latest一致
  • beta: 在latest/stable版本之前的发布版本，用来分享即将发布的新功能。
  • canary: 可以认为是每日构建版本，或者是beta版本之前的准备版本，如果你的包经常更新，或者被很多人依赖，那么你需要发布这个tag。
  • dev: 如果你想通过包仓库的方式测试目前正在开发的包，那么需要发布这个tag。

在某些项目中，开发人员可能会增加自定义的tag，挥着对上面tag的定义做变更。例如，next tag与beta tag是一样的。

尽管我们有很多“标准”的tag，但是真正有实际意义的tag是latest，latest tag是在安装时如果没有指定tag的默认选项。

注意事项

用tag安装和用版本号安装，使用的同一套命令。所以tag得命名不能与版本号的命名重复：

yarn add your-package-name@<version>
yarn add your-package-name@<tag>

如果tag的命名是在semver的规定范围内，那么这个命名也会被拒绝。例如，你不能为tag取名为v2.3，因为这个命名在semver中代表>=2.3.0 <2.4.0。

总的来说，避免用类似于版本的名字来命名tag就好了。

命令

yarn tag add <package>@<version> <tag>

为<package>的指定<version>增加一个<tag>。

yarn tag rm <package> <tag>

从包中删除一个名为<tag>的tag。

注意：在没有备份tag之前，最好不好删除它。

yarn tag ls [<package>]

列出<package>的所有tag。如果没有指定<package>，那么会以你当前的目录作为包的根目录。
