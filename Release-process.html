发布流程

Yarn的新版本发布

  1. 确保当前主分支在 Circle 和 Travis上为绿色
  2. 确保您的本地副本是最新的，然后运行 ./scripts/release-branch.sh. 这将创建0.xx-stable 稳定的分支，CircleCI将自动创建一个GitHub版本
  3. 一旦GitHub版本创建，将 .msi 文件从 该分支的AppVeyor构建如假道Github版本。请注意这不是 自动的，因为我们后续会需要Authenticode签署安装程序（这将在AppVeyor外部完成）
  4. 验证所有弓箭是否连接到版本 (.tar.gz, .deb, .rpm and .msi). 在选中此项之前，请不要继续操作
  5. Bump latest_version 在 _config.yml网站上。这会更新下载URL(/latest.tar.gz 等)以指向新版本。最终这将会被自动化完成(#187)
  6. Debian和CentOS 仓库应该在5分钟内自动更新最新版本 (注意提交)

在现有的Yarn版本上打补丁

  • 切换到已释出的分支 git checkout 0.x-stable, 例如 0.7-stable
  • 优选master分支进行修复
  • 为新的release打一个npm patch版本的tag, 这会创建一个改变了package.json和 tagv0.xx.1的commit
  • Push 到 origin git push origin 0.x-stable --follow-tags

旧的发布流程

这是发布流程的旧版本，供参考（如果任何自动化中断时）。构建发布时一个两个步骤的过程--大多数东西都是建立在Linux上，一些Windows特定的定西（如Windows安装程序）是建立在Windows上的

在Linux上:

  1. ./scripts/release-branch.sh
  2. 构建Debian和RPM包: ./scripts/build-deb.sh
  3. 将 .deb 和 .rpm 文件从 artifacts 目录附加到Github版本

在 Windows上:

  1. 正常构建Yarn (yarn install && yarn run build)
  2. powershell .\scripts\build-dist.ps1
  3. 构建Windows安装程序: yarn run build-win-installer
    • TODO: 为安装程序添加Authenticode签名 (#619)
  4. 构建 Chocolately 包: yarn run build-chocolatey
  5. 将生成的 .msi 文件从artifacts目录附加到Github release中
  6. 上传 Chocolatey 包 (以后我们会做成自动化).
    • 注意: 只有在MSI连接到Github版本时才这样做，因为Chocolatey通过下载链接拉取MSI
    • 还要注意: 在上传Chocolatey软件包后修改MSI将会终端Chocolatey软件包，因为它包含了MSI的hash。确保始终同时更新这两个。
