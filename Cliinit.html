yarn init

交互式的创建或者更新package.json文件。

yarn init

运行这个命令，Yarn会提供一个交互的过程，为你提供一个表单形式的问答卷，来生成package.json。表单中的默认值，如license或者初始版本的信息，在yarn的init-*配置中定义。

下面的例子是在名为testdir运行yarn init命令的结果：

$ yarn init
question name (testdir): my-awesome-package
question version (1.0.0): 
question description: The best package you will ever find.
question entry point (index.js): 
question git repository: https://github.com/yarnpkg/example-yarn-package
question author: Yarn Contributor
question license (MIT): 
success Saved package.json
✨  Done in 87.70s.

生成的package.json如下：

{
  "name": "my-awesome-package",
  "version": "1.0.0",
  "description": "The best package you will ever find.",
  "main": "index.js",
  "repository": {
    "url": "https://github.com/yarnpkg/example-yarn-package",
    "type": "git"
  },
  "author": "Yarn Contributor",
  "license": "MIT"
}

如果你已经有了一个package.json文件，那么运行yarn init命令会以package.json文件中的值作为默认值产生交互式表单问答卷。

假设存在一个package.json，如下所示：

{
  "name": "my-existing-package",
  "version": "0.1",
  "description": "I exist therefore I am.",
  "repository": {
    "url": "https://github.com/yarnpkg/example-yarn-package",
    "type": "git"
  },
  "license": "BSD-2-Clause"
}

交互式问答卷的内容会是这样的

$ yarn init
question name (my-existing-package): 
question version (0.1): 
question description (I exist therefore I am.):
question entry point (index.js): 
question git repository (https://github.com/yarnpkg/example-yarn-package): 
question author: Yarn Contributor
question license (BSD-2-Clause): 
success Saved package.json
✨  Done in 121.53s.
