yarn install

yarn install命令是用来安装项目包的所有依赖。当你从头下载一个新的项目，或者其他协同开发者为项目包新增了一个依赖，你都需要调用这个命令。

如果你之前使用过npm，那么你一定曾经用过--save或者 --save-dev。在Yarn中，这两个参数已经被yarn add和yarn add --dev代替。关于yarn add的更多信息，请查看the yarn add。

yarn install

把package.json文件中列出的所有依赖，安装到node_modules文件夹中。

yarn install --flat

每个依赖仅允许安装一个版本。首次运行命令会提示你为每一个依赖选择版本。你的选择将会被记录了在package.json文件的resolutions选项。

"resolutions": {
  "package-a": "2.0.0",
  "package-b": "5.0.0",
  "package-c": "1.5.2"
}
yarn install --force

将会重新安装所有的依赖，及时那些之前已经被安装过的依赖。

yarn install --har

为安装过程中所有的网络链接输出一个HTTP archive文件。HAR文件通常用来评估网络的好坏，可以通过Google’s HAR Analyzer或者HAR Viewer这类工具分析HAR文件。

yarn install --no-lockfile

安装时不读取或者生成yarn.lock文件。

yarn install --production

使用--production参数，或者当NODE_ENV环境变量被设置为production，Yarn将不会安装package.json中的devDependencies。

Note: --prod是--production的别名。

yarn install --pure-lockfile

安装时不生成yarn.lock文件。
