yarn add

安装依赖以及它自身的依赖。

增加依赖

当你想要使用任何依赖，首先要做的就是要安装它。运行yarn add [package-name]可以把依赖安装到你的项目中。

在安装依赖的同时，也会把这个依赖的相关信息同步到项目中的package.json文件和yarn.lock文件。这样其他开发者在开发的时候，运行yarn或者yarn install命令，就会得到相同的依赖（从依赖版本到依赖结构）。

大部分的依赖包都来自npm registry，并且yarn add命令后面跟随的依赖名字，就是npm仓库中注册的名字。比如，yarn add react将会安装npm仓库的react包。

如果想指定依赖包的版本，可以运行下列命令

  1. yarn add package-name安装最新版本的依赖包
  2. yarn add package-name@1.2.3安装指定版本的依赖包
  3. yarn add package-name@tag安装具体的 “tag” (比如，beta、next或者latest)。

通常，一个依赖包就是一个文件夹，其中包括了代码以及一个package.json文件，package.json文件描述了这个依赖包的所有信息。你通过下面的方法来指明要安装的依赖包或者依赖包的位置：

  1. yarn add package-name从 npm registry中安装依赖包。或者你也可以指定在package.json中指定依赖的其他指向。
  2. yarn add file:/path/to/local/folder从本地文件夹中安装依赖包。这同样用来测试那些没有被正式发布到仓库的包。
  3. yarn add file:/path/to/local/tarball.tgz从本地目录的压缩包安装依赖，这通常是用于在发布之前共享代码。
  4. yarn add <git remote url>从远程git仓库安装依赖包。
  5. yarn add https://my-project.org/package.tgz从远程的压缩包安装依赖。

注意事项

如果你曾经使用过npm，那么你一定知道全局依赖这件事。

对于绝大多数包来说，全局依赖并不是值得推荐的，因为全局依赖是隐性的，模糊的，无法察觉到的。更好的方式是项目中所有的依赖都采用本地依赖的方式，这样更明确，也能保证任何人使用你的项目会得到跟你一样的依赖（从依赖版本到依赖结构）。

CLI工具被定义子在./node_modules/.bin目录。

yarn add <package...>

新增一个依赖，并添加到package.json文件的dependencies属性中。

yarn add <package...> --dev

新增一个依赖，并添加到package.json文件的 devDependencies属性中。

yarn add <package...> --peer

新增一个依赖，并添加到package.json文件的 peerDependencies属性中。

yarn add <package...> --optional

新增一个依赖，并添加到package.json文件的 optionalDependencies属性中。

yarn add <package...> --exact

新增一个明确版本的依赖。如果没有指定exact参数，那么会安装同一个主版本号下最新的release版本，2️而默认的则是版本。例如，yarn add foo@1.2.3将会新增foo的版本是1.9.1，如果使用exact参数，那么yarn add foo@1.2.3 --exact将会新增foo的版本只能是1.2.3。

yarn add <package...> --tilde

新增依赖的版本号，将会是相同次版本号下最新的release版本，而默认的则是会安装同一个主版本号下最新的release版本。例如，yarn add foo@1.2.3 --tilde新增的依赖版本将会是1.2.9 而不是 1.3.0。
